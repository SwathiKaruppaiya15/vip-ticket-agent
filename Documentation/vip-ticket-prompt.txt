--- AGENT 2: vip_agent.py ---
Model: No LLM (pure rule-based + DB lookup)
Purpose: detect VIP status from employee DB + role scoring

VIP scoring rules (deterministic, no LLM):
BASE_SCORES = {
    "C-Level/CEO/CTO/CFO/COO": 50,
    "VP/Vice President": 45,
    "Director": 40,
    "Senior Manager/Manager": 25,
    "Lead/Principal": 15,
}
DEPT_SCORES = {
    "Finance": 20, "Security": 20, "Legal": 18, "HR": 15, "IT": 12,
}
VIP_LEVELS = {"PLATINUM": 80, "GOLD": 60, "SILVER": 40, "STANDARD": 0}
Algorithm:
1. Query MongoDB Employee collection by employee_id
2. If found and has vip_score_override, use that
3. Else compute: role_score + dept_score + (10 if employee.vip_level != STANDARD)
4. Normalize to 0-100 confidence
5. Determine VIP level from VIP_LEVELS thresholds
6. Set state vip_detected = True if confidence >= 40

from langgraph.graph import StateGraph, END
from langgraph.checkpoint.memory import MemorySaver
Build graph:
1. Create StateGraph(AgentState)
2. Add nodes: "intake", "vip_detection", "priority_scoring", "routing", "sla_prediction", "explainability", "notification"
3. Edge flow:
   START → intake → vip_detection → priority_scoring → routing → sla_prediction → explainability → notification → END

   - Invokes graph with config={"configurable": {"thread_id": ticket_id}}
   - Returns final state
   - Logs total processing time with structlog

Endpoints:
POST /auth/register
  - Validate email uniqueness
  - Hash password with bcrypt rounds=12
  - Store User document
  - Return user (no password)
POST /auth/login
  - Verify credentials
  - Generate access_token (JWT, 30min expiry) and refresh_token (JWT, 7 days)
  - Store refresh_token in Redis with key "refresh:{user_id}" and 7-day TTL
  - Update last_login
  - Return {access_token, refresh_token, token_type: "bearer", user}
POST /auth/refresh
  - Validate refresh_token from Redis
  - Issue new access_token
  - Rotate refresh_token (delete old, store new)
POST /auth/logout
  - Delete refresh_token from Redis
  - Add access_token to Redis blocklist with remaining TTL
In app/core/security.py:
- get_current_user dependency: decode JWT, check blocklist in Redis, return User
- require_role(roles: List[UserRole]) dependency factory
- All tokens carry: user_id, role, exp, iat, jti (for blocklist)
